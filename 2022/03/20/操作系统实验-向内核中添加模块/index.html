操作系统实验-向内核中添加模块

发表于 2022-03-20 分类于 实验记录

实验要求

  • 向Linux增加一个系统调用/或内核模块，鼓励程序逻辑的多样化

  • 撰写一个应用测试程序调用该系统调用/测试该内核模块

  • 使用trace/ptrace/strace,或类似的系统跟踪工具来对该测试程序进行跟踪调试

实验环境

  • Vmware Workstation Pro 16.2.1

  • ubuntu-20.04.2.0

  • linux-5.16.11

任务一：向Linux增加一个系统调用或内核模块

配置环境

获取root权限

sudo su

更新系统

sudo apt-get update

安装相关编译程序包

sudo apt-get install libncurses5-dev libssl-dev

sudo apt-get install libelf-dev

sudo apt-get install build-essential openssl

sudo apt-get install zlibc minizip

sudo apt-get install libidn11-dev libidn11

sudo apt-get install flex bison

sudo apt-get install zstd

部署内核

从 The Linux Kernel Archives 下载 linux 内核，这里选择的版本是 linux-5.16.11

下载后移动至虚拟机文件夹/usr/src

进入对应目录，进行解压

sudo tar -xvf linux-5.16.11.tar.xz

添加系统调用

向kernel/sys.c中加入函数

向kernel/sys.c中加入头文件

向arch/x86/include/asm/syscalls.h中添加声明

向arch/x86/entry/syscalls/syscall_64.tbl中添加调用号（这里使用的是335）

编译前的相关配置

删除无关文件

sudo make mrproper

sudo make clean

配置

sudo make menuconfig

编译

sudo make -j8

安装内核

sudo makemodules_install

sudomake install 

重启

reboot

测试是否成功

uname -r

返回当前内核版本号为 5.16.11，即证明成功

错误信息及解决方法

1. 没有规则可制作目标 “debian/canonical-certs.pem”

解决方法：进入/.config,删除CONFIG_SYSTEM_TRUSTED_KEYS、CONFIG_MODULE_SIG_KEY、CONFIG_SYSTEM_REVOCATION_KEYS中的内容

2. 重启后发现内核没有改变

查看当前内核版本

cat /boot/grub/grub.cfg |grep menuentry

修改默认内核启动顺序

修改/etc/default/grub中的GRUB_DEFAULT

更新修改后的grub

update-grub

任务二：撰写一个应用测试程序调用该系统调用/测试该内核模块

编写测试程序

运行测试程序

gcc hello.c

./a.out

返回结果

系统调用成功

任务三：使用trace/ptrace/strace,或类似的系统跟踪工具来对该测试程序进行跟踪调试

这里选择使用 gdb 进行跟踪调试

gcc -g hello.c -o hello_for_debug.out

gdb hello_for_debug.out

r 运行程序

(gdb) r

Starting program: /home/edc/Desktop/hello_for_debug.out system call sys_helloworld return 1 [Inferior 1 (process 2636) exited normally]

b 增加断点，这里选择第十行(return处)增加断点

(gdb) b 10 Breakpoint 2 at 0x5555555551a0: file hello.c, line 10.

bt 查看堆栈调用

(gdb) bt #0 main () at hello.c:10

q 退出

(gdb) q A debugging session is active.

Quit anyway? (y or n) y
